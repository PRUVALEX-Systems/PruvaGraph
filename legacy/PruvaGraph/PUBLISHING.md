# PRUVALEX PruvaGraph — VS Code Marketplace Publishing Guide

This guide walks through the process of publishing PRUVALEX PruvaGraph to the Microsoft VS Code Marketplace.

---

## Prerequisites

Before you can publish, you need:

1. **Microsoft Account** (work, school, or personal)
2. **Azure DevOps Organization** (free tier available at https://dev.azure.com)
3. **Personal Access Token (PAT)** from Azure DevOps
4. **vsce CLI tool** (VS Code Extension CLI) installed globally
5. **The packaged `.vsix` file** (generated by `vsce package`)

---

## Step 1: Set Up Azure DevOps and Personal Access Token

### 1.1 Create an Azure DevOps Organization (if you don't have one)

1. Go to https://dev.azure.com
2. Click **Create organization** or sign in to an existing one
3. Choose a name (e.g., `pruvalex-publishing`)
4. Accept the terms and click **Continue**

### 1.2 Generate a Personal Access Token (PAT)

1. In Azure DevOps, click your **user avatar** (top-right) → **Personal access tokens**
2. Click **+ New Token**
3. Fill in the form:
   - **Name:** `vsce-pruvagraph` (or similar)
   - **Organization:** Select `All accessible organizations`
   - **Expiration:** 1 year (or your preferred duration)
   - **Scopes:** Expand and select **Marketplace** → check both:
     - ✓ Manage (publish)
     - ✓ Acquire
4. Click **Create Token**
5. **Copy the token immediately** — you will not see it again!
   - Store it securely (password manager, `.env` file, etc.)
   - **Never commit it to Git**

---

## Step 2: Register as a Publisher on the Marketplace

### 2.1 Visit the VS Code Marketplace Publisher Registration

1. Go to https://marketplace.visualstudio.com/manage
2. Sign in with your Microsoft Account
3. Click **Create publisher**
4. Enter publisher details:
   - **Publisher Name:** `pruvalex-labs` (matches `extension/package.json` → `publisher` field)
   - **Publisher ID:** Auto-generated from name
5. Accept the terms and click **Create**

### 2.2 Verify Your Publisher

After creation, you should see:
- Publisher name: **pruvalex-labs**
- Publisher ID: **pruvalex-labs** (or similar)
- Marketplace listing: https://marketplace.visualstudio.com/publishers/pruvalex-labs

---

## Step 3: Install vsce (VS Code Extension CLI)

If you haven't installed `vsce` globally:

```bash
npm install -g @vscode/vsce
```

Verify installation:

```bash
vsce --version
```

---

## Step 4: Package the Extension

From the `pruvagraph/extension` directory, run:

```bash
cd pruvagraph/extension
vsce package
```

This generates a `.vsix` file (e.g., `pruvagraph-0.1.0.vsix`) in the current directory.

**Output example:**
```
> vsce package
 INFO  Packaging extension pruvagraph (v0.1.0)...
 INFO  Created pruvagraph-0.1.0.vsix

Generated pruvagraph-0.1.0.vsix
```

---

## Step 5: Publish to the Marketplace

Use your PAT from Step 1.2:

```bash
vsce publish -p YOUR_PAT_TOKEN_HERE
```

**Full example:**
```bash
vsce publish -p vso_your_actual_token_string_here
```

### Expected Output

```
> vsce publish -p vso_...
 INFO  Publishing pruvagraph v0.1.0...
 INFO  Extension published successfully!
 INFO  URL: https://marketplace.visualstudio.com/items?itemName=pruvalex-labs.pruvagraph
```

---

## Step 6: Verify Publication

After publishing, verify the extension appears on the marketplace:

1. Go to https://marketplace.visualstudio.com/search?term=pruvalex
2. Look for **PRUVALEX PruvaGraph** (or search by publisher `pruvalex-labs`)
3. Click the listing to view it

## Updating a Published Extension

To release a new version after changes:

### 6.1 Bump the Version

Edit `pruvagraph/extension/package.json`:

```json
{
  "version": "0.1.1"
}
```

### 6.2 Rebuild and Package

```bash
cd pruvagraph
npm run build
cd extension
vsce package
```

### 6.3 Publish the New Version

```bash
vsce publish -p YOUR_PAT_TOKEN_HERE
```

## Troubleshooting

### "No publisher found"
- **Cause:** Publisher name in `package.json` does not match your registered publisher
- **Fix:** Edit `pruvagraph/extension/package.json` → `publisher` field to match your publisher ID from the marketplace

### "Authentication failed"
- **Cause:** Invalid or expired PAT
- **Fix:** Generate a new PAT in Azure DevOps (Step 1.2)

### "Extension is not uploaded"
- **Cause:** `.vsix` file not found in `pruvagraph/extension` directory
- **Fix:** Run `vsce package` again in the `pruvagraph/extension` directory

### "Version already published"
- **Cause:** You are trying to publish the same version twice
- **Fix:** Bump the version in `package.json` and rebuild

### "Icon not found"
- **Cause:** `assets/icon.png` referenced in `package.json` does not exist
- **Fix:** Either:
  - Create `pruvagraph/extension/assets/icon.png` (128×128 minimum)
  - Remove the `icon` field from `package.json` for now

## Icon Asset (Optional but Recommended)

To add a professional icon:

1. Create `pruvagraph/extension/assets/` directory
2. Add a 128×128 PNG icon as `pruvagraph/extension/assets/icon.png`
3. The icon will appear on the marketplace listing

**If no icon is provided:**
- The marketplace will show a default icon
- Users can still install the extension normally

## Security Best Practices

- ✓ Never commit your PAT to Git
- ✓ Use `.gitignore` to exclude `.env` files containing PATs
- ✓ Rotate PATs periodically (Azure DevOps → Personal access tokens → Delete old tokens)
- ✓ Use separate PATs for different tools/CI/CD pipelines
- ✓ Audit publisher account access regularly
