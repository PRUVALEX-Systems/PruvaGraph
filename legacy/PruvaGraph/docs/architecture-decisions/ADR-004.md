# ADR-004: Dual Extension Architecture

## Status
Accepted

## Context
This repository contains two VS Code extensions:

1. **Root `extension.js`** — The original PruvaGraph extension (v1.0.x).  
   Compiled JavaScript, single-file, distributed as `pruvalex-pruvagraph-*.vsix`.  
   Used by existing users. Mature and stable.

2. **`pruvagraph/` directory** — The new PRUVALEX TypeScript monorepo extension (v0.0.x).  
   Multi-package workspace, follows Hexagonal Architecture + DI + Repository patterns.  
   Currently in active development.

## Decision
Both extensions coexist during the transition period. The root extension serves existing
users while PRUVALEX is developed. Once PRUVALEX reaches feature parity, the root 
`extension.js` will be deprecated and users migrated.

## Migration Path
1. PRUVALEX reaches feature parity with PruvaGraph (all 5 modules stable)
2. PRUVALEX is published to VS Code Marketplace as `pruvagraph`
3. PruvaGraph extension shows deprecation notice pointing to PRUVALEX
4. Root `extension.js` removed in v2.0.0

## Consequences
- Developers must be aware that `npm run build` in root is for PruvaGraph, 
  `npm run build` in `pruvagraph/` is for PRUVALEX
- CI runs both builds independently
- `.vscodeignore` for each is separate
